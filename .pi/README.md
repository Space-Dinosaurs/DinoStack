# DinoStack - Pi Coding Agent Adapter

Native adapter for the current [Pi coding agent](https://www.npmjs.com/package/@earendil-works/pi-coding-agent), which discovers project resources from `.pi/`.

This is distinct from the legacy `.omp/` adapter for oh-my-pi.

## Quick start

Project-local use works without install when running `pi` from this repository:

```bash
bash .pi/build.sh
pi
```

Then use:

```text
/skill:agentic-engineering
/brief <topic>
/implement-ticket <ticket>
/wrap
```

Pi discovers:

- skills from `.pi/skills/`
- prompt templates from `.pi/prompts/`
- context files from `AGENTS.md` and `CLAUDE.md`

## Install globally

```bash
git clone https://github.com/Space-Dinosaurs/DinoStack.git ~/agentic-engineering
bash ~/agentic-engineering/.pi/install.sh
```

Optional non-interactive activation settings:

```bash
bash .pi/install.sh --mode=opt-in --profile=default
bash .pi/install.sh --mode=opt-out --profile=strict
```

Global install writes:

- `~/.pi/agent/skills/agentic-engineering/SKILL.md`
- `~/.pi/agent/skills/agentic-engineering/METHODOLOGY.md`
- symlinks from `~/.pi/agent/skills/agentic-engineering/{commands,references,rules,agents}` to this repo's `content/`
- symlinks from `~/.pi/agent/prompts/*.md` to `.pi/prompts/*.md`
- shared activation config at `~/.claude/agentic-engineering.json`

## Uninstall global install

```bash
bash .pi/uninstall.sh
```

## Adapter layout

```text
.pi/
  build.sh
  install.sh
  uninstall.sh
  README.md
  prompts/                         # Pi prompt templates generated from content/commands/*.md
  skills/agentic-engineering/
    SKILL.frontmatter.yaml
    SKILL.md                       # generated from content/SKILL.md
    METHODOLOGY.md                 # generated from content/sections/*.md
    commands -> ../../../content/commands
    references -> ../../../content/references
    rules -> ../../../content/rules
    agents -> ../../../content/agents
```

## Generated `.pi` files

The following project-local files are generated by other Pi packages/tools and intentionally ignored:

- `.pi/agents/supervisor.md`
- `.pi/gsd/**`
- `.pi/taskplane.json`
- `.pi/git/**`
- `.pi/npm/**`

## Verification

Run from the repo root (`~/agentic-engineering`):

```bash
cd ~/agentic-engineering && bash .pi/build.sh
test -f ~/agentic-engineering/.pi/skills/agentic-engineering/SKILL.md
test -f ~/agentic-engineering/.pi/skills/agentic-engineering/METHODOLOGY.md
test -L ~/agentic-engineering/.pi/skills/agentic-engineering/commands
test -f ~/agentic-engineering/.pi/prompts/brief.md
cd ~/agentic-engineering && git check-ignore -q .pi/gsd/VERSION
cd ~/agentic-engineering && git check-ignore -q .pi/agents/supervisor.md
cd ~/agentic-engineering && git check-ignore -q .pi/taskplane.json
cd ~/agentic-engineering && git status --short
```
