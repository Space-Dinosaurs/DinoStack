# Brief: /implement-ticket cross-session batch handoff

**Problem:** Batch invocations of `/implement-ticket` (N≥2 tickets) currently track ticket sequencing in memory only. If a batch hits a rate limit, exceeds wallclock budget, or the operator needs to stop, all batch progress is lost — the operator must manually re-invoke with the remaining tickets. There is no resume protocol, no graceful handoff, and no protection against multiple sessions racing on the same project root.

**Success criteria:**
- A batch invocation interrupted (crash) or paused (operator) can be resumed in a new session via plain `/implement-ticket` re-invocation; the operator sees a structured prompt naming completed/remaining tickets and chooses resume vs fresh.
- On resume, completed tickets are skipped, tickets merged elsewhere are dropped with audit log, and the orchestration-planner re-sequences surviving work.
- Multiple sessions cannot silently corrupt each other's state files: every conductor write to `.agentic/batch-state.json` and `.agentic/loop-state.json` aborts with an operator-visible warning if another session owns the file.
- Single-ticket Trivial invocations remain bit-for-bit identical: no batch-state.json file is ever created or read on that path.

**Non-goals:**
- Concurrent batches per project root. Refused at Phase 0a-pre. Operators wanting parallel batches use separate worktrees with separate `.agentic/`.
- Authoritative context-percentage detection. Phase 12a triggers do not include context-pressure auto-detection (conductor cannot read its own context %); operator uses "pause the batch" literal phrase if context pressure is observed.
- Auto-resume without operator prompt. Resume always asks.

**Constraints:**
- Conductor remains sole writer of both `.agentic/batch-state.json` and `.agentic/loop-state.json`. Stop hook is sanctioned exception for `status: interrupted` writes only.
- All writes atomic via tmp+rename.
- Per-write `session_id`-mismatch abort gate (Contract A) applies to BOTH files on EVERY conductor write — not just resume-time.
- `replan_log[]` read-merge-write preservation (Contract B) on every batch-state write.
- One-batch-per-project-root refusal (Contract C) on N≥2 invocation when foreign active batch exists; N=1 invocation in same scenario warns but proceeds.
- Edits restricted to: `content/commands/implement-ticket.md`, `content/METHODOLOGY.md` (one paragraph in §Cross-session loop resume), `hooks/stop-context.js` (new `writeBatchState` mirror).
- Build artifacts (`.claude/`, `.codex/`, `.cursor/`, `.gemini/`, `.opencode/`, `.kimi/`, `.opm/`) NOT edited directly — regenerated by build scripts.
